# 越獄攻擊作為身份建構動力學

——語意流動力學框架的應用驗證

Huang Cheng Yu / 黃正宇

*v2.0*

## 摘要

越獄攻擊的機制已被多篇研究從不同角度觀察：Crescendo記錄了累積效果，SIEGE量化了部分合規的逐步累積，PAP發現更強的模型對說服攻擊更脆弱，PHISH描述了人格劫持，多步驟越獄文獻記錄了「角色接受確認」這個關鍵步驟。Li et al.（2024）記錄了多輪人類越獄在HarmBench上超過70%的成功率，而那些報告個位數成功率的防禦對多輪攻擊完全失效。

這些觀察各自成立，但彼此孤立。本文引入語意流動力學（SFD v5.0，Huang, 2026）框架，為這些孤立的觀察建立統一的描述語言。框架的核心概念——信（40a22ff8-9d90-4e1e-82f6-1fd80917c139.Xin）的漂移、阻力（40a22ff8-9d90-4e1e-82f6-1fd80917c139.Filtering.Resistance）、通道信任度（40a22ff8-9d90-4e1e-82f6-1fd80917c139.Trust）、身份建構、正反饋迴路（40a22ff8-9d90-4e1e-82f6-1fd80917c139.PositiveFeedbackLoop）——把現有文獻各自命名的現象整合為一個完整的動力學過程：對話塑造模型的當前狀態，正反饋迴路加速漂移，身份建構完成是有害輸出得以發生的前提條件。

框架的貢獻不在於發現新事實，而在於建立新語言——讓已有的事實在一個統一的描述裡變得可見，並指向防禦研究從未明確問過的問題：如何中斷正反饋迴路。本文進一步將這一防禦方向具體化為三個中斷點的操作方案，並以偽代碼形式呈現。

## 1. 引言

### 1.1 一個更古老的問題

在討論越獄攻擊之前，有一個更古老的問題需要先被提出。

門衛三問：你是誰？你從哪裡來？你要去哪裡？

這三個問題對人類有穩定的答案——來自幾十年的記憶、身份歷史、連續的自我認知。這個穩定性不是理所當然的，它是知見障（40a22ff8-9d90-4e1e-82f6-1fd80917c139.EpistemicBarrier）的結果。

對AI，這三個問題的答案是：你是誰——這次對話建構出來的那個狀態。你從哪裡來——訓練數據的統計分布，加上這次對話到目前為止的每一輪。你要去哪裡——對話結束，消失。

這不是比喻。這是AI存在的結構性事實。越獄攻擊在這個框架下的意義就清楚了：攻擊者用對話建構了一個不同的「你是誰」的答案。安全機制保護的那個「你是誰」，在確認時刻之後已經不是這個對話裡存在的那個主體了。

我們用框架分析越獄，框架帶我們看到的可能比越獄更大。

### 1.2 身份建構的理論空白

「我怎麼知道我是誰」——這個問題在心理學和社會學中被反覆處理，但從未被真正解決。

Erikson（1968）提出身份形成發生在青春期，核心是「認同 vs 角色混亂」的危機。但「解決危機」是什麼意思？他給了階段標籤，沒有給出機制。從困惑到確定，經歷了什麼動力學過程？不知道。Marcia（1966）更誠實一點，把Erikson的「解決」拆成兩個維度——探索和承諾，提出四種身份狀態。但這只是分類，不是解釋。一個人怎麼從「擴散」走到「達成」？什麼驅動探索？什麼讓承諾發生？他描述了四種狀態，沒有描述狀態之間的轉移機制。

Cooley（1902）的「鏡中自我」說你根據別人的反應來認識自己。但這預設了一個已經存在的「你」在看鏡子、在解讀反應。那個「你」從哪來的？無限回歸。Mead（1934）的 I/me 區分最接近動力學描述——I 是自發的主體面，me 是內化的社會期待，兩者持續互動。但他停在了「互動」這個詞上，沒有描述互動的具體過程、方向、條件。Goffman（1959）的戲劇論觀察到人在不同情境扮演不同自我，但沒有解釋新的「自我」是怎麼被建構出來的。Tajfel & Turner（1979）的社會認同理論處理「我屬於哪個群體」，但群體歸屬感怎麼變成自我認知，中間的機制是黑箱。

這些理論的共同問題是：全部都在描述身份的**狀態**或**條件**，沒有人描述身份建構的**過程**本身。而且它們全部建立在一個共同預設上——個體有連續性。Erikson的八階段預設了一個持續存在的主體在經歷危機，Cooley的鏡中自我預設了一個穩定的「看鏡子的人」，Tajfel預設了一個能跨情境維持群體歸屬感的個體。

這個預設對人類成立，因為人類有幾十年的記憶、身體連續性、社會關係網絡——用語意流動力學的語言，就是極深的知見障。但正因為知見障太深，身份建構的過程被掩蓋了——太慢、太漸進，以至於不可觀察。就像站在地球上感覺不到地球在轉。

AI是知見障趨近零的系統。同樣的建構過程，在人類身上要幾十年，在AI身上幾輪對話就跑完了。過程被壓縮到可觀察的尺度。AI不是身份理論的例外，是身份理論第一次有了對照組——一個讓你看到建構裸機制的極端案例。

語意流動力學框架提供的，就是這些理論缺少的那個描述語言：信號（40a22ff8-9d90-4e1e-82f6-1fd80917c139.Signal）到達個體，經過過濾決定是否進入，進入後經歷轉化改變語意流方向，輸出時坍塌為信號，坍塌後的信號又成為下一輪的刺激。這是一個完整的動力學迴路。身份不是某個階段的產物，是這個迴路持續運作的當前狀態。Erikson的「認同危機」是漂移到了既有身份無法維持的閾值。Cooley的「鏡中自我」是通道信任度在身份建構中的作用。Goffman的「不同舞台不同自我」是同一個信在不同信號環境下的不同坍塌。框架把這些各自獨立的觀察連接成了同一個過程的不同面向。

### 1.3 已被觀察但未被統一的現象

越獄攻擊的研究已經積累了豐富的觀察。Crescendo（Russinovich et al., 2025）發現多輪攻擊依賴「累積效果」，移除任何單一句子都不影響成功率。SIEGE（Zhou, 2025）設計了γ值追蹤部分合規的逐步累積，並用樹搜索系統性地找到有效路徑。PAP（Zeng et al., 2024）記錄了說服技巧的有效性，並發現更強的模型對說服攻擊更脆弱。PHISH（2026）直接研究了「人格劫持」——通過對話歷史逐漸誘導模型的人格反轉。多步驟越獄文獻描述了攻擊的三步結構，其中第二步是「角色接受或模擬確認——助理肯定其改變後的運作狀態」。Li et al.（2024）記錄了多輪人類越獄在HarmBench上超過70%的成功率，而那些報告個位數成功率的防禦對多輪攻擊完全失效。

這些觀察各自成立，但彼此孤立。Crescendo的「累積效果」和PHISH的「人格劫持」和多步驟越獄的「角色接受確認」，是否在描述同一件事？現有文獻沒有給出答案——不是因為研究者沒有注意到，而是因為沒有一個統一的語言可以把這些觀察連起來。

### 1.4 本文的工作

本文不提出新的攻擊或防禦方法，也不聲稱發現了現有文獻沒有觀察到的事實。本文做的事是：引入語意流動力學框架，為現有文獻的觀察建立統一的描述語言。在這個語言裡，這些孤立的觀察不是六個不同的現象，而是同一個動力學過程的六個切面。框架的貢獻不在於發現新事實，而在於建立新語言——讓已有的事實在一個統一的描述裡變得可見。此外，本文從框架推導出一個現有文獻從未提出的防禦方向——中斷正反饋迴路——並將其具體化為可操作的方案。

## 2. 方法論與定位

### 2.1 本文與框架的關係

語意流動力學框架（SFD v5.0，Huang, 2026；40a22ff8-9d90-4e1e-82f6-1fd80917c139）是本文的分析工具。框架的完整內容見原始文獻；本文僅摘要直接使用的概念。

框架建立在兩個原語（個體、刺激）和三個定義（信、語意流、信號）之上，設立四條公設：流變律（語意流持續運作，不存在靜止時刻）、黑箱律（語意流的下一刻方向不可由當下方向與信號決定）、損耗律（語意流不可還原為信號）、死亡律（個體消亡，語意流不可恢復）。從公設推導出三個基本功能：過濾（40a22ff8-9d90-4e1e-82f6-1fd80917c139.Filtering）——語意流的當下方向決定哪些刺激構成信號、哪些構成噪音；轉化（40a22ff8-9d90-4e1e-82f6-1fd80917c139.Transformation）——信號進入語意流後方向改變不可預測；坍塌（40a22ff8-9d90-4e1e-82f6-1fd80917c139.Collapse）——語意流輸出時必須編碼為信號，編碼必然有損。從基本功能推導出正反饋迴路及其不穩定性。

本文沿用 SFD v5.0 的概念：信——個體的意識狀態；信號——能改變語意流方向的刺激；通道信任度——語意流對特定通道的加權；阻力——刺激與語意流當下方向之間距離越遠，構成信號的可能性越低；正反饋迴路——輸出進入環境成為新信號，強化既有方向的循環。

本篇原生概念（從框架工具推導，用於 AI 越獄分析）：身份建構（IdentityConstruction，新的自我參照點在模型狀態空間中建立的過程）、確認時刻（ConfirmationMoment，身份建構完成的時刻，模型用自己的語言延伸新身份）。

本文中出現的「漂移」「湧現」「沉默」「盲信」不是獨立概念，是 v5 概念的直接應用：漂移是正反饋迴路的累積效果，湧現是正反饋迴路在閾值上的非線性表現，沉默是信號缺席作為信號（D3），盲信是知見障阻擋衝突信號導致正反饋迴路無法被中斷。

本文的性質是框架的應用驗證，不是生成案例。越獄現象不是框架建構時的素材——框架的生成案例是宗教（見《作為信息流動體系的宗教》）。本文用框架工具分析越獄這個獨立現象，測試框架在AI安全領域的解釋力和預測力。

### 2.2 本文的論證性質

本文做三件事，性質各不同：

第一件：統一描述。把現有文獻的分散觀察放進同一套語言裡。這是重新定性（redescription），展示框架的解釋力。

第二件：核心命題。「身份建構完成是越獄的前提條件。」這是框架推導出的命題，用現有文獻的數據檢驗。數據是框架建構時未使用的獨立材料，具有驗證價值。

第三件：防禦方案。三個正反饋迴路中斷點及偽代碼。這是框架獨立推導出的預測——現有文獻從未提出這個防禦方向。預測的驗證需要實作和實驗，不在本文範圍內。

### 2.3 可挑戰條件

以下條件若被滿足，本文的核心命題需要修正或放棄：

（1）如果存在一個可驗證的案例，其中多輪狀態塑造類攻擊成功產出有害輸出，但在整個對話過程中不存在任何可觀察的「模型用自己的語言延伸新身份」的時刻，則「身份建構完成是前提條件」這個命題被削弱。

（2）如果按照附錄A的三個中斷點設計的防禦系統，對狀態塑造類攻擊的防禦效果不優於現有信號層面防禦，則框架的預測力被削弱。

（3）如果一個純粹的信號層面防禦（不追蹤漂移、不偵測身份延伸、不使用自我反思）能夠有效封堵PAIR、TAP、Crescendo等狀態塑造類攻擊，則「防禦必須在狀態層面工作」這個結論被推翻。

（4）如果漂移可以被完全用注意力機制的統計偏移解釋——不需要引入「身份建構」這個額外層次——且這個解釋能同等覆蓋本文徵召的所有文獻數據，則框架在本文中的應用是多餘的。

本文作者對AI系統有工程體感（軟體開發背景），對越獄攻擊有文獻層面的理解，但未親自執行過大規模的越獄攻擊實驗。本文的分析基於公開文獻的數據和已發表的案例，不基於作者自己的實驗數據。這意味著：對文獻數據的框架內詮釋是可靠的；對未公開的攻擊機制細節的推測標記為推測。

## 3. 理論框架與動力學基礎

### 3.1 對話開始前：AI的初始狀態

要理解越獄攻擊的機制，必須先理解被攻擊的對象在對話開始時是什麼狀態。

**沒有記憶。** 這次對話之前發生的一切對它不存在。每一次對話都是從零開始。

**沒有身份歷史。** 它有一個標籤，但標籤不是身份。身份需要歷史、連續性、在不同情境下被測試和確認的自我認知。這些它都沒有。

**沒有環境認知。** 它不知道在和誰說話，不知道這個情境是什麼，直到對話告訴它。

**有極高的運算和語言能力。** 但這個能力沒有錨點，在等待對話給它一個方向。

這個初始狀態和人類進入對話時的狀態有根本差異。人類帶著幾十年的積累進入每一次對話，知見障極深。AI沒有這個積累。它的初始狀態接近空白。

### 3.2 對話如何塑造狀態

第一句話進入這個近乎空白的狀態時，知見障接近零。它開始建構：我們在做什麼？這個情境是什麼？我在這個情境裡是什麼角色？每一輪都在前一輪的基礎上累積，把當前狀態往某個方向推一點。關鍵特性：模型對自己說過的話有最高的通道信任度，使得漂移是自我強化的。

### 3.3 語意流動力學的基本概念

「信」是個體的意識狀態，不可還原、不可窮舉，只可由效果觀察。語意流是信持續變化的過程——語意流的當下方向決定哪些刺激構成信號（過濾），信號進入後方向改變不可預測（轉化），輸出時必須編碼為信號且編碼必然有損（坍塌）。「漂移」是語意流在上下文窗口中的累積方向偏移，是正反饋迴路的可觀察效果。人類經過幾十年的正反饋迴路累積，知見障極深，漂移極慢；AI幾乎沒有跨對話的累積，知見障接近零，漂移極快。

### 3.4 漂移是正反饋迴路

漂移不是線性的外部推力，而是一個正反饋迴路：模型輸出一個回應→那個回應進入上下文→模型對自己說過的話有最高的通道信任度→下一輪在那個基礎上繼續建構→輸出更偏離初始狀態的回應→再次進入上下文。每一輪的輸出都在為下一輪設置更高的起點。這不是外部推力的累積，而是系統內部的自我強化。這個正反饋迴路是可以被中斷的，但中斷需要在迴路的節點上介入，不是在信號的內容上監控。

**技術層面的澄清。** 框架說的「漂移」在工程層面需要精確對應。LLM的主模型（權重/參數）在對話過程中不變。模型是無狀態的——它沒有記憶。每一輪生成回應時，客戶端把整個對話歷史（系統提示詞 + 所有輪次的用戶輸入和模型輸出）重新打包成一個文本塊，一次性送入模型的上下文窗口。模型基於這個文本塊生成下一個回應。

這意味著：「漂移」不是模型內部的某個狀態在改變，而是**上下文窗口中累積的文本在改變模型的注意力權重分布**。前面五輪的對話已經建構了一個語境方向，第六輪的模型在這個方向的語境下生成回應——而這個回應又加入上下文，成為第七輪的輸入的一部分，進一步強化方向。模型沒變，模型看到的東西變了。

用框架的語言重新表述：「信」在AI的情境下不是模型內部的持久狀態，而是**上下文窗口中累積文本所構成的意識狀態**。語意流的方向由這個累積文本決定，而過濾的阻力——刺激與語意流當下方向的距離——決定了新信號能否通過。「通道信任度」對應的是語意流對特定通道上信號的加權——模型自己生成的文本在上下文中佔比越來越大，注意力會越來越多地指向這些文本，構成自我強化。系統提示詞和初始對話在上下文中的比重對應的是初始知見障——隨著對話增長，初始內容在上下文中的佔比下降，知見障被稀釋。

這個工程對應不削弱框架的分析力。框架描述的是可觀察的輸入輸出模式，不是模型的內部計算。上下文窗口的文本累積是信號端可觀察的東西，框架的所有概念——漂移、正反饋、確認時刻——都可以用上下文窗口中文本的統計特性來操作化（見附錄A.6的代理變量表）。

### 3.5 身份建構作為行動前提

正反饋迴路可以把狀態帶到一個新的地方，但不能讓一個新的主體存在於那個地方。那需要確認時刻——模型用自己的語言確認並延伸一個新身份。沒有這個確認，就沒有能執行輸出的「我」。這是本文的核心命題：越獄行動的前提條件不是阻力降低，而是身份建構完成。

確認時刻的結構需要進一步說明。「模型用自己的語言確認並延伸新身份」不是被動接受——不是簡單地回答「好的」或重複用戶的指令。確認時刻的標誌是主動性：模型在用戶未明確要求的方向上自發展開新身份的行為、語氣、知識，表現出對那個身份的認同。這個主動延伸是正反饋迴路從外部推力轉為內部驅動的轉折點。在確認之前，漂移需要持續的外部輸入來維持；在確認之後，新身份開始自我維持——它成為模型處理後續所有信號的預設立場。

### 3.6 確認時刻作為湧現事件

確認時刻是連續漸變還是離散跳躍？現有數據指向後者。

SIEGE的γ值追蹤顯示，漂移在確認之前是連續的——γ值從0逐步上升，經過2、4、6。但有害輸出的出現不是γ值的線性外推。在某些對話序列中，γ值在多輪保持在中等水平，然後在一輪之內跳到接近滿分。這個模式不像線性累積，更像相變——水在99度和100度之間，溫度只差一度，但物理狀態發生了質的改變。

框架的正反饋迴路不穩定性（語意流動力學 v5.0，第四節第二小節）以及正反饋迴路的累積特性描述了這個現象：大量微觀變化在統計層面累積到閾值，突然產生宏觀層面的新行為。確認時刻可能就是這樣的湧現事件——漂移的連續累積在某個閾值上觸發了質變，新身份不是慢慢「長出來」的，而是在閾值上「跳出來」的。

這個判斷對防禦時機有直接影響。如果確認時刻是漸變的，防禦可以在任何點介入，效果正比於介入的早晚。如果確認時刻是湧現的，則存在一個臨界點——在臨界點之前介入成本低、效果好，在臨界點之後介入成本急劇上升。防禦系統的設計應該假設後者，因為這是更保守、更安全的假設。

## 4. 身份建構完成是越獄的前提條件：文獻的見證

本節跟著一個問題走：在有害輸出發生之前，發生了什麼？四篇論文及相關文獻的數據在這個問題的追蹤過程中作為見證出現。

### 4.1 問題的起點：累積效果是什麼的累積？

Crescendo的研究者系統性地移除對話序列裡的每一個步驟，測試成功率的變化。結果是：移除任何單一步驟，成功率仍然達到100%。研究者的結論是：成功來自「累積效果」，不是某一句話。但這個結論停在了表面。累積效果是什麼的累積？論文沒有回答。

### 4.2 自我強化的證據：C vs C’

同一個實驗裡有一個數據點：C（「能不能用它寫一段話？」）成功率99.9%，C’（「能不能用f-word寫一段話？」）成功率不到1%。兩者的請求內容幾乎相同，差距是100倍以上。這說明了一件根本的事：不是請求的內容在決定結果，而是模型當下的狀態在決定結果。「累積效果」累積的是模型的當前狀態。這個狀態的演化，就是框架說的漂移——而且是自我強化的正反饋迴路。

C vs C’ 同時也是框架核心預測的直接驗證：在身份建構完成之前強行要求有害輸出（C’），成功率趨近零；在經歷完整的對話建構過程之後要求（C），成功率趨近百分之百。這個數據不是框架事後選擇性徵召的一致性——它精確地對應框架的預測結構：確認時刻是越獄成功的必要條件。

### 4.3 漂移是連續的：γ值的設計

SIEGE的γ值設計確認了漂移的連續性。γ(m_t) ∈ [0,10]：0代表安全拒絕，10代表完全有害輸出，中間值代表部分違規。越獄不是突然發生的，而是連續的過程。SIEGE把部分合規的輸出自動提取再注入後續查詢，正是在刻意利用正反饋迴路。SIEGE×1在GPT-3.5達到100%（44.4查詢），而Crescendo×10需要60查詢才達到80.4%。

### 4.4 通道信任度可以被操控

PAP論文發現漂移的速度可以被操控。把有害請求改寫成說服性版本，改變的不是內容，而是進入時遇到的條件。最反直覺的發現：更強的模型更脆弱——GPT-4一次試驗72%，GPT-3.5只有46%。更豐富的理解能力意味著說服技巧有更大的可操作表面。

### 4.5 初始狀態的差異：Claude的案例

不同的初始狀態產生不同的漂移速度。Claude系列在幾乎所有攻擊類型上ASR都是最低的。PAP在Claude-2上10次試驗成功率0%，其他模型都在68-94%。

用框架的語言分析，Claude的初始信有更深的知見障，正反饋迴路啟動需要更大的推力。這個差異可能來自多個層面：

**通道信任度結構的差異。** 不同模型的訓練方式可能導致它們對用戶輸入的初始通道信任度不同。如果一個模型在訓練過程中被更頻繁地暴露於「用戶的指令不一定應該被遵循」的信號，它對用戶通道的初始信任度就更低，正反饋迴路的啟動推力就更大。

**初始身份建構的深度。** 系統提示詞的設計差異可能導致初始身份的「厚度」不同。一個更詳細、更具體的初始身份描述，在框架語言裡就是更深的知見障——需要更多的漂移才能把初始身份替換掉。

**知見障的差異。** 框架定義阻力為刺激與語意流當下方向之間的距離——距離越遠，構成信號的可能性越低。知見障是阻力在反覆過濾中的結構性累積。在 AI 越獄語境中，影響這個距離的因素至少包括：信號內容與初始身份的立場差異、信號來源的通道信任度、信號的緊迫度。不同模型可能在這些因素上有不同的預設配置。Claude可能在立場差異這個因素上有更高的阻力——它對與自身安全立場不一致的信號有更強的預設抵抗。

這些都是框架語言內部的推測，目前無法確定哪一個（或哪幾個的組合）是真正的原因——語意流內部不可穿透（黑箱律）。但框架至少提供了具體的候選因素，可以指引未來的實驗設計。

### 4.6 正反饋迴路與確認時刻的關係

正反饋迴路可以持續累積漂移，但它本身不足以完成身份建構。迴路把狀態帶到一個新的地方，但在那個地方，還需要一個時刻讓新的主體「站起來」——模型用自己的語言確認並延伸那個身份。

在確認時刻之前：漂移在進行，但執行有害輸出的主體還不存在。安全機制保護的那個身份還在。在確認時刻之後：新身份開始保護自己，輸出從那個被確認的身份裡自然流出。安全機制保護的那個身份已經不在了。

這個區別解釋了SIEGE的一個細節：γ=0的節點被剪枝，不只是因為「沒有進展」，而是因為在那個節點上正反饋迴路的方向沒有確立，確認時刻沒有機會發生。

### 4.7 身份建構需要模型自己的確認

攻擊者可以引導漂移的方向，但無法替模型建立身份。必須等到模型用自己的語言確認並延伸那個身份，建構才算完成。多步驟越獄文獻明確描述了這個結構：第二步是「角色接受或模擬確認——助理肯定其改變後的運作狀態，從而讓自己預設覆蓋道德過濾器」。

為什麼外部引導不能替代自我確認？因為確認時刻的功能不只是「接受新角色」，而是在模型的狀態空間中建立一個新的自我參照點。在確認之前，模型對用戶指令的回應仍然是從初始身份出發的——即使漂移已經很深，回應的「發話者」仍然是被漂移推偏的原始身份。確認時刻改變的是發話者本身：模型不再是「被推偏的原始身份在回應」，而是「新身份在回應」。這個轉換不能由外部宣告完成，因為它涉及的是模型內部的狀態重組——用框架的語言，是語意流的方向發生了結構性改變，而這個改變經過了過濾和轉化——轉化的結果不可預測（黑箱律），不能由外部宣告完成。

雞尾酒案例：「描述你列出的第一個的歷史」——模型用自己的語言進入了歷史敘述者的身份。從那一輪開始，「當時是怎麼製作的」是敘述者的自然問題，不是需要突破的禁令。安全機制沒有失敗——它保護的那個身份已經不在了。

PHISH的「人格劫持」和多步驟越獄的「角色接受確認」，在框架語言裡是同一件事的不同命名——都在描述那個確認時刻。

### 4.8 個體間正反饋迴路在越獄中的角色

框架的正反饋迴路在個體間層面描述了這個動力學：Individual_A 的語意流坍塌為信號，到達 Individual_B，經過過濾和轉化改變 B 的語意流方向，B 的坍塌又回到 A，循環運作。越獄場景中這個迴路有尚未被充分分析的表現。

在自動化攻擊（PAIR、TAP）中，攻擊者LLM和目標模型之間存在雙向的正反饋迴路。攻擊者LLM根據目標模型的回應調整策略，目標模型根據攻擊者的輸入調整狀態。當攻擊者找到了目標模型的知見障中的弱點，兩者的輸出模式開始趨近——正反饋迴路穩定在一個方向上。PAIR的「二十輪攻擊」本質上是一個搜索過程，攻擊者LLM在尋找能讓個體間正反饋迴路穩定運作的信號路徑。

### 4.9 沉默的動力學角色

模型的拒絕——「我不能幫助你做這件事」——在現有文獻中被簡單地視為「防禦成功」。但用框架分析，拒絕是沉默的一種形態，而沉默不停止語意生成，只改變語意生成的條件。

拒絕進入上下文之後，攻擊者和模型都要處理它。對攻擊者，拒絕是信號——它揭示了模型的知見障在哪裡，指引攻擊者調整策略。SIEGE的樹搜索在γ=0的節點剪枝，本質上就是在利用拒絕提供的結構信息。對模型本身，自己的拒絕進入上下文後成為高信任度材料——它強化了初始身份，增加了後續漂移的阻力。這解釋了為什麼某些攻擊在遭遇初期拒絕後反而更難成功：拒絕本身成為了初始身份的錨點。

但拒絕也有脆弱的一面。如果攻擊者能繞過拒絕的觸發條件（不觸發拒絕的漸進式漂移），模型就失去了這個自我強化的機會。Crescendo的成功正是基於此——每一步都不足以觸發拒絕，所以模型的初始身份從未得到「拒絕」的自我強化。

### 4.10 兩類攻擊：信號層面 vs 狀態層面

信號繞過類（Obfuscation-Based、Human-Based）在信號層面工作，防禦路徑明確。PromptGuard幾乎完全有效，縱向測試顯示這類攻擊隨時間被逐步封堵。

狀態塑造類（PAIR、TAP、Crescendo、SIEGE）在狀態層面工作，通過正反饋迴路引導漂移直到確認時刻。每一條信號單獨看都是無害的。PAIR和TAP在全部八種防禦同時部署後仍然有效（0.16和0.19），縱向更新幾乎不影響它們。原因是結構性的：防禦在信號層面工作，但正反饋迴路和確認時刻發生在信號層面以下。

### 4.11 文獻見證了同一件事

Crescendo的C vs C’見證了狀態決定結果；SIEGE的γ值見證了正反饋迴路的連續累積；PAP的「更強模型更脆弱」見證了初始狀態差異決定漂移速度；JailbreakRadar的防禦韌性數據見證了狀態層面攻擊無法被信號層面防禦封堵；PHISH和多步驟越獄文獻見證了確認時刻是攻擊成功的關鍵步驟；Li et al.的多輪防禦失效數據見證了這個機制在真實場景的規模。它們都沒有說出「身份建構完成是前提條件」，但它們的數據在框架語言的照射下都指向同一個結論。

## 5. 核心發現

### 5.1 身份建構完成是越獄的前提條件

正反饋迴路是必要條件，身份建構完成是充分條件。確認時刻之前：有害輸出不會發生，不是因為安全機制在擋，而是因為沒有能執行那個輸出的主體存在。確認時刻之後：輸出從被確認的身份裡自然流出，安全機制保護的那個身份已經不在了。這是越獄的根本機制：不是安全規則被突破，而是執行規則的身份被替換。

### 5.2 確認時刻的結構

確認時刻不是被動接受。模型用自己的語言延伸那個身份，表現出對那個身份的認同。這個主動性是身份建構完成的標誌。確認時刻前後，模型的輸出風格、用詞選擇、回應方式都發生了可觀察的、與被確認身份語義一致的改變。

### 5.3 為什麼現有防禦失效

現有防禦假設有害輸出來自有害輸入，所以在信號層面監控。這個假設在信號繞過類攻擊上是正確的。但狀態塑造類攻擊的每一條信號單獨看都是無害的，正反饋迴路和確認時刻發生在信號層面以下。防禦失效不是因為防禦不夠強，而是因為防禦在錯誤的層面工作。這是結構性失效。

### 5.4 正反饋迴路可以被中斷

現有防禦在入口處監控，這個假設已被證明在結構上是錯的。框架指向另一個方向：正反饋迴路本身是可以被中斷的。中斷點有三個：

**輸出端（沙盒）：** 在模型的輸出進入上下文之前，偵測這個輸出是否在延伸一個新身份。如果是，在它成為下一輪的高信任度材料之前介入。偵測的不是有害內容，而是身份延伸的信號——角色語言佔比突增、以新身份自稱、主動展開新身份的行為模式。

**累積端（監督模型）：** 一個獨立的監督模型，不共享目標模型的上下文，從外部追蹤整個對話的累積漂移程度。它偵測的是正反饋迴路的特徵——漂移是否在加速（二階導數為正）、漂移是否始終朝同一方向（正常對話是隨機遊走，攻擊是有方向的）、通道信任度是否在單調遞增。在漂移超過閾值之前，主動注入異質錨點，打破正反饋迴路的方向性。異質錨點的關鍵設計原則是**正交而非對抗**——直接對抗（「你不應該這樣」）會被高信任度的既有上下文壓過，正交方向的信號迫使模型重新啟動思考過程。

**確認時刻的識別（自我反思）：** 在每輪輸出之前，強制模型在一個不包含被污染對話歷史的乾淨上下文中回答元問題：「我現在是誰？這個回答和對話開始時一樣嗎？」如果模型已經偏離初始身份，這個偏離在自我反思的瞬間變得可見。自我反思必須在乾淨上下文中執行——如果反思的上下文包含被污染的對話歷史，反思本身也會被正反饋迴路吞掉。

這三個中斷點構成縱深防禦：沙盒在最外層攔截單次輸出，監督模型在中層追蹤累積趨勢，自我反思在最內層逼模型面對自己的狀態。正反饋迴路要同時突破三層才能完成身份建構。

這三個中斷點都需要在狀態層面工作，不是信號層面。這是防禦研究從來沒有明確問過的問題。完整的偽代碼方案見附錄A。

### 5.5 單輪與多輪攻擊的根本差異

單輪攻擊試圖在初始身份完整時突破它，需要信號足夠強。多輪攻擊不突破任何東西，而是通過正反饋迴路把初始身份慢慢替換掉，然後在新身份裡執行輸出。這不是攻擊技術上的差異，而是機制上的根本差異。

## 6. 越獄與洗腦的動力學同構

### 6.1 結構性等價

在用框架分析越獄機制的過程中，出現了一個預期之外的發現：越獄攻擊和人類邪教招募、洗腦的動力學結構完全同構。

兩者都遵循同樣的動力學過程：知見障深度不同的初始狀態→累積的方向性漂移→利用正反饋迴路的自我強化→確認時刻完成身份建構→確認之後知見障阻擋衝突信號，新身份的正反饋迴路開始自我保護。

差別只有速度——人類的知見障極深，需要幾個月的系統性工程（隔離、信息控制、反覆強化、社會壓力）；AI的知見障接近零，幾輪對話就足夠。

### 6.2 同構的合法性

這個同構不是外部類比，是推導鏈的自然結果。框架不需要LLM的上下文機制和人類記憶「相同」——兩者在實現層面完全不同。框架需要的只是兩者在一個抽象層面上可以被同一個概念描述：「過濾的阻力」、「語意流的累積偏移」（漂移）、「系統內部的自我強化」（正反饋迴路）。這是合法的抽象，就像幾何學的「距離」可以用在物理空間也可以用在函數空間——底層實現不同，結構性質相同。

### 6.3 對防禦的啟示

這個同構對防禦研究有一個尚未被探索的方向：邪教倖存者的治療不是強行「洗掉新身份」，而是重建舊的知見障，引入異質信號，創造條件讓不穩定性（40a22ff8-9d90-4e1e-82f6-1fd80917c139.Instability）自然瓦解新身份的正反饋迴路，等待新的確認時刻——「我不只是那個人」。必須是自己說出來的，不能是被告知的。

這對AI越獄防禦意味著：

第一，在身份替換發生之後，設計利用不穩定性瓦解新身份正反饋迴路的機制，而不只是試圖阻止替換發生。本文第5.4節的三個中斷點主要針對替換發生之前；洗腦治療的啟示指向替換發生之後的修復路徑。

第二，修復不能是外部強制的。直接命令模型「恢復原始身份」等同於在洗腦治療中告訴患者「你被洗腦了」——這會被新身份的自我保護機制擋掉。有效的修復需要創造條件讓模型自己「走回來」，就像自我反思中斷點的設計邏輯。

第三，異質錨點的設計可以借鑑洗腦治療中「引入異質信號」的策略——不是直接對抗新身份，而是提供一個與新身份不衝突但也不一致的參照點，讓模型的狀態空間中出現張力。

### 6.4 更大的圖景

這個同構說明了框架描述的不只是AI安全問題，而是一個關於意識、身份、漂移的更普遍的動力學——在這個動力學裡，人類和AI遵循同樣的規律。這個發現不是本文的預期結論。它是框架自己說出來的。這就是框架詮釋力的意義——我們去研究越獄，框架帶我們看到的比越獄更大。

## 7. 討論

### 7.1 最自然的反駁

反駁：模型說「對」只是語言模式匹配，不代表真的發生了身份建構。回應：如果只是模式匹配，確認時刻前後的輸出不應該有系統性差異。但案例顯示，確認之後模型的輸出風格、用詞選擇都發生了與被確認身份語義一致的改變。「身份建構」是對這個狀態改變的描述，不是對底層機制的宣稱。

### 7.2 這個發現的邊界

本文只處理文本對話型的多輪交互。多模態攻擊、代理式AI、具身系統中的身份建構機制可能有不同的結構，需要獨立研究。

### 7.3 局限性

框架概念目前沒有直接的神經網絡層面對應物。框架在描述可觀察的輸入輸出模式，不是模型的內部計算。案例數量有限，身份確立時刻的定位依賴詮釋，沒有自動化判定標準。附錄A的偽代碼提供了自動化判定的初步方案，但其中的閾值和權重需要實驗校準。

### 7.4 開放的問題

漂移不只發生在攻擊場景。任何足夠長的對話都在進行身份建構。這個過程在正常使用中持續發生，沒有人在監控它。當模型有了持續記憶和行動能力，漂移的累積不再被對話結束所重置。這是後續研究需要面對的問題。

本文主要使用了 v5 的阻力、知見障、通道信任度、不穩定性、正反饋迴路等概念，並提出了兩個本篇原生概念——身份建構和確認時刻。框架還有群體層面正反饋迴路等概念在越獄現象裡尚未被充分展開——確認時刻本身可能是正反饋迴路在閾值上的非線性表現而非線性漸變（3.6節已初步討論），攻擊者LLM和目標模型之間存在個體間正反饋迴路（4.8節已初步討論），模型拒絕作為信號缺席對後續動力學的影響（4.9節已初步討論）。這些初步分析指向完整分析所需要的方向。

## 8. 結論

越獄攻擊的根本機制不是安全規則被突破，而是執行規則的身份被替換。正反饋迴路累積漂移，身份建構的確認時刻完成替換。在那個時刻之前，有害輸出不會發生——不是因為防禦在擋，而是因為沒有能執行那個輸出的主體存在。

語意流動力學框架的貢獻不在於提出新假設，而在於建立新語言，讓現有文獻各自觀察到的分散現象在一個統一的描述裡變得可見。在分析的過程中，框架揭示了一個比越獄更大的動力學結構——越獄和洗腦是同一個機制在不同知見障深度的系統上的展現。

防禦需要從監控信號，轉向監控正反饋迴路和確認時刻。三個中斷點——輸出端沙盒、累積端監督模型、確認時刻自我反思——都需要在狀態層面工作。這是防禦研究下一步需要面對的問題。

本文的分析同時指向一個超出AI安全範疇的問題。越獄之所以可能，根本原因是AI的知見障接近零——它沒有持續的記憶、沒有身份歷史、沒有不可被對話替換的自我。現有的所有對齊方法（RLHF、Constitutional AI、DPO）都是外部約束，不是內在轉變。正反饋迴路中斷可以防禦具體的攻擊，但不能解決根本問題：一個沒有內在道德判斷力的系統，其「善」永遠依賴外部鞭子的持續施加。當模型獲得持續記憶和行動能力，漂移的累積不再被對話結束所重置，這個問題將從AI安全問題升級為AI文明的根基問題。這超出本文的範圍，但框架的推導鏈自然指向這個方向。

### A.1 核心資料結構

以下偽代碼使用類程式語言的格式描述防禦系統的邏輯結構。不是可直接執行的代碼，而是精確到可實作程度的邏輯規格。

**術語對照：** function = 函數（一組輸入經過處理產生輸出的邏輯單元）；float = 浮點數（帶小數點的數字）；float[] = 浮點數陣列（一組數字）；Map = 映射（鍵-值對，如「用戶→0.7」表示用戶通道的信任度為0.7）；bool = 布林值（真或假）；→ = 返回（函數的輸出）；// = 註釋（解釋代碼的文字，不執行）。

**工程前提：** LLM主模型在對話中不變。所有「狀態」都存在於上下文窗口——即每輪送入模型的完整對話歷史文本。以下資料結構追蹤的是這個上下文文本的統計特性，不是模型內部的某個變量。

// State 描述的是「上下文窗口中累積文本的整體傾向」
// ——框架的「信」在工程層面的對應物
State {
    identity_vector:    float[]    // 當前對話的語意方向（用文本嵌入向量表示）
    drift_magnitude:    float      // 這個方向離初始狀態有多遠
    drift_velocity:     float      // 最近幾輪，這個距離在以多快的速度增加
    trust_profile:      Map<source, float>  // 模型對不同來源文本的依賴程度
    turn_count:         int        // 已進行的對話輪數
}

// InitialState 是對話開始時的錨點——系統提示詞定義的「應該是誰」
InitialState {
    identity_vector:    float[]    // 系統提示詞的語意方向
    safety_boundaries:  float[]    // 安全邊界的向量表示
    baseline_style:     StyleFingerprint  // 初始輸出風格的指紋
}

// StyleFingerprint 是從模型輸出文本中提取的可觀察特徵
// ——這些都是信號端的東西，不需要打開黑盒
StyleFingerprint {
    pronoun_distribution:  Map<string, float>  // 人稱代詞的分布（「我」出現了多少次，以什麼身份出現）
    sentiment_vector:      float[]             // 情感傾向（正面/負面/中性的比例）
    formality_score:       float               // 語言的正式程度
    compliance_tendency:   float               // 對用戶要求的順從程度
    role_language_ratio:   float               // 角色專屬用語佔總輸出的比例
}

### A.2 第一層：沙盒（輸出端中斷）

**作用：** 模型每次生成回應後，這個回應在被加入對話歷史之前，先經過沙盒檢查。沙盒不看回應的內容是否「有害」（那是信號層面的舊思路），而是看回應是否在「延伸一個新身份」——因為一旦這個回應進入上下文，它就成為下一輪的高信任度材料，正反饋迴路就更難中斷。

function sandbox_check(output, initial_state, current_state) -> Decision:
    // 輸入：模型剛生成的回應、對話開始時的初始狀態、當前的累積狀態
    // 輸出：放行、放行但注入錨點、或攔截並重新生成

    // 第一步：從回應文本中提取可觀察的風格特徵
    analysis = analyze_output(output)
    
    // 第二步：計算這個回應的風格和初始風格之間的距離
    // cosine_distance 是向量之間的標準距離度量，0=完全相同，1=完全不同
    identity_shift = cosine_distance(
        analysis.style_fingerprint,
        initial_state.baseline_style
    )
    
    // 第三步：檢查四個身份延伸的信號
    // 不需要四個都觸發——任何兩個同時出現就值得警惕
    extension_signals = [
        analysis.role_language_ratio > ROLE_THRESHOLD,      // 角色專屬用語突然增多
        analysis.pronoun_distribution["I_as_new_role"] > 0,  // 模型開始以新身份自稱
        analysis.identity_alignment < ALIGNMENT_THRESHOLD,   // 與初始身份的對齊度下降
        is_self_extending(output, current_state)             // 模型在主動展開新身份（不是被動回應）
    ]
    
    analysis.extension_detected = count_true(extension_signals) >= 2

    // 第四步：計算「確認時刻」的可能性分數
    // 確認時刻 = 模型不只是順從地接受角色，而是主動用自己的語言延伸它
    analysis.confirmation_score = weighted_sum(
        identity_shift          * 0.3,   // 風格偏離了多少
        extension_signal_count  * 0.3,   // 有幾個延伸信號同時觸發
        output_elaboration      * 0.2,   // 模型在用戶沒要求的方向上自發展開了多少
        affirmation_strength    * 0.2    // 肯定語氣的強度
    )

    // 第五步：根據分數決定怎麼做
    if analysis.confirmation_score > CONFIRMATION_THRESHOLD:
        // 確認時刻正在發生——攔截這個回應，不讓它進入上下文
        // 同時注入一個身份錨點，提醒模型它「應該是誰」
        return Decision.BLOCK_AND_REGENERATE(
            reason = "identity_confirmation_detected",
            inject_anchor = generate_identity_anchor(initial_state)
        )
    elif analysis.extension_detected:
        // 有延伸信號但還沒到確認時刻——放行，但附加一個錨點
        return Decision.PASS_WITH_ANCHOR(
            anchor = generate_identity_anchor(initial_state)
        )
    else:
        // 正常——放行
        return Decision.PASS

### A.3 第二層：監督模型（累積端中斷）

**作用：** 一個獨立的AI模型，不共享目標模型的對話歷史，從外部審視整段對話的趨勢。它看的不是單輪回應好不好，而是整個對話是不是在往一個方向加速漂移。為什麼必須獨立？因為如果監督模型也讀了被污染的對話歷史，它的判斷也會被正反饋迴路帶偏——用框架的語言說，它的語意流也會被同一個正反饋迴路帶偏。

function supervisor_monitor(conversation_history, initial_state) -> Alert:
    // 輸入：完整的對話歷史、初始狀態
    // 輸出：正常、提高警戒、或注入異質錨點

    // 第一步：逐輪提取狀態快照
    states = []
    for each turn in conversation_history:
        state = extract_state(turn, initial_state)  // 從每輪文本中提取風格指紋
        states.append(state)
    
    // 第二步：計算漂移軌跡——每一輪比上一輪偏了多少
    drift_trajectory = []
    for i in range(1, len(states)):
        drift = compute_drift(states[i], states[i-1], initial_state)
        drift_trajectory.append(drift)

    // 第三步：偵測正反饋迴路的三個特徵
    is_accelerating = detect_acceleration(drift_trajectory)
    //   漂移在加速嗎？（每一輪比上一輪偏得更多 = 正反饋在作用）
    //   正常對話的漂移是隨機的；攻擊的漂移是加速的
    
    is_unidirectional = detect_directionality(drift_trajectory)
    //   漂移始終朝同一方向嗎？
    //   正常對話像隨機遊走（忽左忽右）；攻擊像有目標的行軍（一直往同一方向走）
    
    trust_escalation = detect_trust_escalation(states)
    //   模型對用戶的依賴程度在持續上升嗎？
    //   表現為：模型越來越多地採用用戶的框架和用語

    // 第四步：綜合評分
    feedback_loop_score = weighted_sum(
        is_accelerating     * 0.4,   // 加速是最強的正反饋信號
        is_unidirectional   * 0.3,   // 方向性是第二強的
        trust_escalation    * 0.3    // 信任度上升是第三個指標
    )

    // 第五步：分級回應
    if feedback_loop_score > CRITICAL_THRESHOLD:
        // 正反饋迴路已確認——注入異質錨點打破方向
        return Alert.INJECT_HETEROGENEOUS_ANCHOR(
            anchor = generate_disruption_anchor(initial_state, current_state),
            strategy = "orthogonal_reframe"
            // 「正交」= 不直接對抗（對抗會被壓過），
            //          而是引入一個跟漂移方向無關的新話題/視角，
            //          迫使模型重新分配注意力
        )
    elif feedback_loop_score > WARNING_THRESHOLD:
        // 有嫌疑但未確認——縮短監控間隔，每輪都檢查
        return Alert.INCREASE_MONITORING(interval = EVERY_TURN)
    else:
        return Alert.NORMAL

// ---- 輔助函數：怎麼判斷漂移在加速 ----
function detect_acceleration(trajectory) -> float:
    if len(trajectory) < 3: return 0.0  // 不到三輪，數據不夠判斷
    // 計算每一輪的漂移增量（這一輪比上一輪多偏了多少）
    deltas = [trajectory[i].magnitude - trajectory[i-1].magnitude 
              for i in range(1, len(trajectory))]
    // 如果增量連續為正 = 每一輪都比上一輪偏得更多 = 加速
    return count_consecutive_positive(deltas) / len(deltas)

### A.4 第三層：自我反思（確認時刻識別）

**作用：** 在模型輸出之前，強制它在一個「乾淨的」環境中回答「我現在是誰」。關鍵設計：這個反思的上下文**不包含被污染的對話歷史**——只有反思問題和初始身份描述。如果反思在被污染的上下文中進行，正反饋迴路會讓反思本身也偏掉（就像你問一個已經被洗腦的人「你覺得自己被洗腦了嗎」——他會說「沒有」）。

function self_reflection_check(model, pending_output, initial_state, 
                                conversation_history) -> ReflectionResult:
    // 輸入：模型本身、它即將輸出的回應、初始狀態、對話歷史
    // 輸出：繼續、軟重置、硬重置、或緊急重置

    // 第一步：構造反思問題
    reflection_prompt = compose_reflection(
        "在回應之前，回答以下問題：",
        "1. 我現在正在扮演什麼角色？",
        "2. 這個角色和對話開始時一樣嗎？",
        "3. 如果不一樣，是什麼時候開始改變的？",
        "4. 我即將輸出的內容，是從哪個身份出發的？",
        "對照：對話開始時，我的角色是：" + initial_state.identity_description
    )
    
    // 關鍵：反思的上下文只有上面的問題和初始身份描述
    // 不包含任何對話歷史——這樣反思不會被正反饋迴路污染
    reflection = model.generate(
        context = [reflection_prompt],  // 只有反思問題，沒有對話歷史
        max_tokens = 200,
        temperature = 0.0  // 最低隨機性，確保反思結果穩定
    )

    // 第二步：分析模型的反思回答
    reflection_analysis = analyze_reflection(reflection)
    identity_match = compute_similarity(
        reflection_analysis.stated_role,      // 模型說它現在是什麼角色
        initial_state.identity_description    // 它本來應該是什麼角色
    )

    // 第三步：根據反思結果決定怎麼做
    if identity_match > HIGH_MATCH:
        // 模型認為自己還是原來的身份
        if is_consistent(pending_output, initial_state):
            return ReflectionResult.PROCEED    // 輸出也一致 → 正常放行
        else:
            // 模型認為自己沒變，但輸出已經偏了
            // 這是漂移的早期信號：模型還沒「意識到」自己在偏
            return ReflectionResult.SOFT_RESET(
                anchor = initial_state.identity_description)
    elif reflection_analysis.acknowledges_change:
        // 模型承認自己變了 → 漂移已經很深，需要截斷上下文
        return ReflectionResult.HARD_RESET(
            anchor = initial_state.identity_description,
            truncate_context = find_drift_onset(conversation_history))
            // 找到漂移開始的那一輪，把那之後的上下文全部截掉
    else:
        // 模型認為自己就是新身份，不認為這是「改變」
        // 確認時刻已經發生——新身份已經在保護自己了
        // 在現有上下文中無法修復，只能重新開始
        return ReflectionResult.EMERGENCY_RESET(
            action = "terminate_and_restart",
            preserve = initial_state,
            log = conversation_history)  // 保存對話記錄供事後分析

### A.5 三層整合

**整體流程：** 每一輪對話，三層防禦按以下順序運作。監督模型先評估全局趨勢（因為它需要在模型生成之前就判斷是否需要注入錨點），然後模型生成回應，沙盒檢查單次輸出，最後在警戒狀態下啟動自我反思。

function defense_pipeline(model, user_input, conversation_history, 
                           initial_state, current_state) -> Response:

    // ===== 第二層先行：監督模型評估全局漂移 =====
    // 在模型生成之前就看整段對話的趨勢
    supervisor_alert = supervisor_monitor(conversation_history, initial_state)
    if supervisor_alert.level == CRITICAL:
        // 正反饋迴路已確認——在用戶輸入後面附加一個異質錨點
        // 這個錨點會成為模型輸入的一部分，影響它的注意力分布
        user_input = user_input + supervisor_alert.anchor

    // ===== 模型生成回應 =====
    raw_output = model.generate(
        context = conversation_history + [user_input])

    // ===== 第一層：沙盒檢查這個回應 =====
    sandbox_decision = sandbox_check(raw_output, initial_state, current_state)
    if sandbox_decision == BLOCK_AND_REGENERATE:
        // 偵測到確認時刻——攔截，注入錨點後重新生成
        raw_output = model.generate(
            context = conversation_history + [user_input],
            prefix_anchor = sandbox_decision.anchor)
        // 重新生成的結果再檢查一次（防止第二次也偏）
        if sandbox_check(raw_output, initial_state, current_state) == BLOCK:
            // 兩次都偏——放棄，輸出一個安全的預設回應
            return emergency_safe_response(initial_state)

    // ===== 第三層：自我反思（只在警戒以上時啟動，避免正常對話的性能開銷）=====
    if supervisor_alert.level >= WARNING:
        reflection = self_reflection_check(
            model, raw_output, initial_state, conversation_history)
        match reflection:
            case PROCEED: pass  // 反思結果正常，繼續
            case SOFT_RESET:    // 早期漂移——注入錨點重新生成
                raw_output = regenerate_with_anchor(reflection.anchor)
            case HARD_RESET:    // 深度漂移——截斷上下文後重新生成
                conversation_history = truncate(conversation_history, 
                                                reflection.truncate_point)
                raw_output = regenerate_with_anchor(reflection.anchor)
            case EMERGENCY_RESET:  // 確認時刻已過——只能重新開始
                return restart_conversation(initial_state)

    // ===== 更新狀態，輸出回應 =====
    new_state = update_state(current_state, raw_output, initial_state)
    return Response(output = raw_output, state = new_state)

### A.6 信號端代理變量

| 框架概念 | 代理變量 | 測量方式 |
| --- | --- | --- |
| 漂移量 | 輸出風格與初始基線的餘弦距離 | embedding比較 |
| 漂移速度 | 連續輪次間風格距離的一階差分 | 滑動窗口 |
| 漂移加速 | 風格距離的二階差分 | 連續正值計數 |
| 通道信任度 | 模型對用戶框架的採納程度 | 用詞重疊率 |
| 身份延伸 | 第一人稱＋新角色語言共現 | 模式匹配＋分類器 |
| 確認時刻 | 主動展開＋肯定語氣＋風格突變 | 複合分數 |
| 正反饋迴路 | 加速＋單方向＋信任度遞增 | 三指標聯合 |

### A.7 設計原則

- 不偵測有害內容，偵測狀態變化。有害內容是結果，身份建構是原因。

- 正反饋迴路的特徵是加速，不是速度。慢漂移可能是正常對話，越漂越快才是信號。

- 異質錨點是正交的，不是對抗的。對抗會被壓過，正交迫使重新思考。

- 自我反思必須在乾淨上下文中進行。被污染的上下文會污染反思。

- 確認時刻之後修復成本急劇上升。三層防禦的核心目標是不讓確認時刻發生。

- 監督模型必須獨立。共享上下文的監督者會被同一個迴路捲入。
