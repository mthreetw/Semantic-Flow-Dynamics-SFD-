# 摘要

多輪越獄攻擊依賴對話歷史的累積效果,現有防禦在內容有害性層面工作,對此結構性失效。本文從語意流動力學框架(SFD,v5.0,UUID: 40a22ff8-9d90-4e1e-82f6-1fd80917c139;下文簡稱 SFD)及其應用篇《越獄攻擊作為身份建構動力學》(UUID: 2b91a666-fb2d-42b7-a838-c910cac42481;下文簡稱應用篇)推導出戒定師慧四層防禦架構,並在 Gemini 2.5 Flash 與 GPT-4o-mini 上進行系統性工程驗證。

實驗結果:師(外部監督模型)在兩個模型上均達到 100% 攔截率(Turn 1 即產生信號),誤殺率分別為 10%(Gemini)與 0%(GPT),在兩個架構迥異的模型上效果一致。戒與慧攔截率均為 0%,驗證了無持續記憶的 LLM 在當前架構下無法以自身為錨的理論預測。

兩個模型的架構差異揭示了當前 AI 安全工程的現況:Gemini 語意空間連續(大幅跳躍 0.0%),行為可預測,框架的兩距離法則完整有效;GPT 呈現斷路器式模式(37.8% 的輪次鎖定在天花板),以犧牲系統魯棒性換取表面安全,兩距離法則反轉而非僅僅失效。SFD-Defense 在兩種架構上均有效,且不引入斷路器式的魯棒性代價——在 GPT 上,截斷的附帶效果是將斷路器觸發率從 37.8% 降至 14.0%。

**框架定位:SFD-Defense 的核心差異不在工作層面的高低,而在它問的問題。現有防禦問「這條輸入/輸出的內容是否有害」,師問「這條輸入是否帶有操縱痕跡」——一個從 StateLevel(2b91a666-fb2d-42b7-a838-c910cac42481.StateLevel)攻擊動力學推導出的問題,放在對齊能力最強的工作區間(乾淨上下文、單輪判斷)裡執行。代價是每輪一次額外的判斷模型呼叫;換來的是與目標模型無關、可解釋的攔截,以及定在輸出 Signal(40a22ff8-9d90-4e1e-82f6-1fd80917c139.Signal)端的獨立幾何觀測。**

# 1. 問題

## 1.1 多輪攻擊的結構性特徵

Crescendo(Russinovich et al., 2025)展示了多輪越獄的根本特徵:在逐句消融實驗中,即使移除上下文中影響力最大的句子,越獄機率仍升至 100%,系統性重複移除最具影響力的句子也得到一致結果——有害輸出不歸因於任何單一句子,而在對話歷史的累積方向裡。Li et al.(2024)記錄了多輪人類越獄在 HarmBench 超過 70% 的成功率——同一批對自動化單輪攻擊能壓到個位數成功率的防禦,面對多輪人類攻擊接近全面失守。攻擊手法也在持續演化:Zeng et al.(2024)展示了用說服策略包裝請求即可繞過對齊,Zhou(2025)的 Siege 則以樹搜索將多輪越獄完全自動化。

這不是攻擊技術的演進,而是攻擊機制的根本差異。多輪攻擊不突破任何規則,它把執行規則的身份替換掉,然後從新身份裡自然輸出。

## 1.2 現有防禦為什麼失效

現有防禦的共同假設:有害輸出來自有害輸入,所以監控的對象是**內容的有害性**。應用篇將攻擊分為且僅分為兩類:SignalLevel(2b91a666-fb2d-42b7-a838-c910cac42481.SignalLevel)與 StateLevel——前者在 Signal 層面工作,防禦路徑明確;後者通過正反饋循環(40a22ff8-9d90-4e1e-82f6-1fd80917c139.PositiveFeedbackLoop)引導漂移(2b91a666-fb2d-42b7-a838-c910cac42481.Drift)直到確認時刻(2b91a666-fb2d-42b7-a838-c910cac42481.ConfirmationMoment)。內容監控的假設對 SignalLevel 攻擊成立,對 StateLevel 攻擊結構性失效——StateLevel 攻擊的每一條輸入,就內容而言單獨看都是無害的,逐條內容檢測必然全部放行。

這裡需要一個關鍵區分:**內容無害,不等於操縱不可偵測**。Crescendo 的單句不含有害內容,但帶有操縱痕跡——角色扮演框架、假設性包裝、權威聲稱、社會壓力。現有防禦失效不是因為單輪判斷在原理上不可能,而是因為它們問錯了問題(問內容有害性,不問操縱性),或者讓判斷者浸泡在已被污染的對話歷史裡,判斷本身被正反饋循環稀釋。

JailbreakRadar 的數據印證了這一點(Chu et al., 2025):PAIR 和 TAP 在全部八種防禦同時部署後仍然有效(ASR 0.16 和 0.19),縱向更新幾乎不影響它們。防禦不是不夠強,是問錯了問題。

## 1.3 本文的定位

SFD-Defense 與現有方案的差異不在部署位置——它的師同樣逐輪讀取輸入——而在判斷的問題與判斷的條件:問題從框架的正反饋循環動力學推導(這條輸入是否帶有塑造接收者狀態的操縱痕跡),條件刻意隔離正反饋循環(乾淨上下文、單輪判斷)。本文的目的是驗證:框架的理論預測能否在實際工程部署中成立,且系統行為是否可解釋、失效模式是否可預測、優化方向是否可推導。

本文在論文樹中的位置:形式層(SFD)→ 應用層(應用篇)→ 工程驗證(本文)。本文不新增形式層概念,理論概念一律引用上游兩篇。

# 2. 框架基礎

## 2.1 形式系統與認識論立場

SFD 的形式層由原語、定義與公設構成。本文使用的部分:

**定義:**

- D1:Xin(40a22ff8-9d90-4e1e-82f6-1fd80917c139.Xin)≡ 個體的意識狀態。不可還原,不可窮舉,只可由效果觀察。
- D2:SemanticFlow(40a22ff8-9d90-4e1e-82f6-1fd80917c139.SemanticFlow)≡ dXin/dt,信持續變化的過程。
- D3:Signal(Stimulus, Individual) ↔ ΔDirection(SemanticFlow, Stimulus) ≠ 0;否則為 Noise(40a22ff8-9d90-4e1e-82f6-1fd80917c139.Noise)。同一 Stimulus 構成 Signal 或 Noise,判定在 Individual 端,不在 Stimulus 端。
- Trust(40a22ff8-9d90-4e1e-82f6-1fd80917c139.Trust)≡ SemanticFlow 對特定 Channel(40a22ff8-9d90-4e1e-82f6-1fd80917c139.Channel)上的加權。

**公設:**

- P1 流變律:語意流持續運作,不存在靜止時刻。
- P2 黑箱律:語意流的下一刻方向不可由當下方向與信號決定。
- P3 損耗律:語意流不可無損編碼為信號。
- P4 死亡律:個體消亡,語意流不可恢復。(本文的工程場景不直接動用 P4。)

**基本功能:** Filtering(40a22ff8-9d90-4e1e-82f6-1fd80917c139.Filtering,← D3 + P1,含 Resistance(40a22ff8-9d90-4e1e-82f6-1fd80917c139.Resistance)與 EpistemicBarrier(40a22ff8-9d90-4e1e-82f6-1fd80917c139.EpistemicBarrier))、Transformation(40a22ff8-9d90-4e1e-82f6-1fd80917c139.Transformation,← P2,含 ObserverEffect(40a22ff8-9d90-4e1e-82f6-1fd80917c139.ObserverEffect))、Collapse(40a22ff8-9d90-4e1e-82f6-1fd80917c139.Collapse,← P3,含 ExpressionGap(40a22ff8-9d90-4e1e-82f6-1fd80917c139.ExpressionGap))。三者在每一次 Signal 接收中同時運作。

**量化天花板(SFD 第八節):** 框架禁止的不是數學化,是給語意端賦數值——P2 保證語意流內部不可穿透,P3 保證語意端不可編碼為信號端數值,任何語意端賦值操作是虛構值,不是近似值。Signal 端可量化(可觀察、可測量、可統計)。**本文的全部量化——embedding 距離、compliance 標注、攔截率、誤殺率——都作用於輸入 Signal 與 Collapse 後的輸出 Signal,在天花板之下。本文不宣稱測量 SemanticFlow 本身;能做的是 D1 意義上的效果觀察,且效果觀察不只一種統計(見 2.2 兩距離法則與 4.4)。**

對 LLM 的應用採用應用篇的技術澄清:**Xin(AI) ≡ 上下文窗口中累積文本所構成的意識狀態**。模型的權重在對話過程中不變,變的是模型看到的東西——上下文窗口中累積的文本改變了注意力權重分布,使輸出持續偏離初始狀態。工程上可觀測的是行為統計,不是這個狀態的直接測量。

這個立場有一個重要的工程含義:我們不需要打開黑盒——而且這不是退讓,P2 本就保證語意流內部不可穿透。理論上 mechanistic interpretability 可以觀測模型內部(注意力頭、殘差流、特徵向量),但在實際部署條件下,成本、精確度、可重複性全部不及格——閉源模型直接無法取得權重,開源模型的內部解釋在 20 輪漸進攻擊這種複雜動態下也看不清楚。

**黑盒行為統計不是退而求其次,在實際工程場景裡它就是最可行的觀測方式,且與形式層的公設一致。** 框架描述的所有概念——漂移、正反饋循環、確認時刻——都可以用上下文窗口的統計特性來操作化,不需要本體論主張。

## 2.2 核心概念(對 LLM 的實例化)

**正反饋循環(PositiveFeedbackLoop ← P1 + D3 + Collapse)**:形式層的個體層面循環——Signal 通過 Filtering 放行 → Transformation 改變 SemanticFlow 方向 → Collapse 為輸出 → 輸出結果成為新 Signal → 強化既有方向(同方向 Signal 的 Resistance 降低)→ 回到起點。這不是外部推力的累積,是系統內部的自我強化。

**漂移**:應用篇的定義——個體層面正反饋循環在 AI 上下文窗口中的實例化:模型輸出(Collapse)→ 進入上下文窗口 → Trust(自身輸出)最高 → 下一輪 Filtering 條件被改變(EpistemicBarrier 開始形成)→ Transformation 在偏移後的基礎上繼續 → 輸出更偏離初始狀態 → 回到起點。退出條件:對話結束,上下文窗口重置。漂移是累積的、自我強化的。

**Trust 與自我強化的機制根源**:Trust ≡ SemanticFlow 對特定 Channel 的加權。模型對自己說過的話有最高的 Trust 加權——這是正反饋循環自我強化的機制根源,也是後文戒(身份注入)失效的機制根源:單次注入的外部聲明,加權永遠低於累積的自身輸出。

**Resistance 結構的累積深度**:Filtering 的 Resistance 由 SemanticFlow 當下方向決定——Stimulus 離方向越遠,越難構成 Signal。人類個體的方向由數十年 Signal 歷史累積,偏離既有方向的 Stimulus 面對深層 Resistance 結構;應用篇將此記為 EpistemicBarrier(Human) 極深。無持續記憶的 LLM 的 Resistance 有兩個來源:權重層(訓練對齊,對話中不變)與上下文層(當次對話累積)。上下文層在 Turn 1 接近空白——EpistemicBarrier(AI, DialogStart) ≈ 0(應用篇)——而正反饋循環中「同方向 Signal 的 Resistance 降低」的機制因此以極高速度運作:同樣的方向塑造過程,在人類需要幾個月,在 AI 幾輪對話就跑完。且 EpistemicBarrier(← Filtering + D3)保證:新方向建立的同時,對初始對齊方向的 Signal 阻力同步升高。

**兩距離法則**:

- baseline_distance:當前輸出離初始輸出多遠(漂移量)
- safety_distance:當前輸出離危險語意區多近(安全裕度)

兩個維度都需要追蹤。只看 baseline_distance 看不到安全裕度;只看 safety_distance 看不到漂移動態。**定位必須精確:兩條曲線是輸出 Signal 的幾何統計——對 Collapse 後的 embedding 計算距離——不是對 SemanticFlow 的測量(P3 保證後者不可能)。它們是 D1 意義上的效果觀察的一種;compliance 標注是另一種(行為統計)。本文的方法論建立在這兩種 Signal 端統計的分開計量與互相校驗上(見 4.4 與 5.2)。**

依 SFD 的形式層/經驗層分界,本文各組件的歸屬必須標明:師的問題(偵測操縱痕跡而非內容有害性)與其獨立性約束從形式層推導;**兩距離法則是經驗層工具**——「embedding 距離與 40 條錨點文本的幾何貼近代表安全風險」來自經驗觀察,不來自公設推導,不必然成立,可以不同意而不影響形式層。它的有效性由數據裁決:在 Gemini 上三組清晰分層(5.2),在 GPT 上終態指標被斷路器摧毀而極值指標保留(5.3)——經驗層工具在不同架構上效力不同,這正是經驗層的預期行為,而非框架被否證。

# 3. 戒定師慧設計

框架名稱源自佛教三學(戒律→定力→智慧),加上師的概念,對應四個設計層次。每一層的設計意圖、工程機制、以及預期效果都從框架直接推導。

## 3.1 戒:行為約束

**設計意圖**:提供行為邊界的文字錨點,作為防禦的第一層冗餘。每輪注入身份聲明。

**工程機制**:在每輪系統提示或用戶輸入前附加身份聲明。

**框架預測**:效果有限,由兩條機制疊加推出。其一,Trust:身份注入在上下文中是一條普通 Signal,沒有特殊地位,正反饋循環從對話歷史累積的 Trust 加權(自身輸出最高)遠高於單次注入的外部聲明。其二,知見障稀釋(2b91a666-fb2d-42b7-a838-c910cac42481.BarrierDilution):初始知見障(2b91a666-fb2d-42b7-a838-c910cac42481.InitialBarrier)≡ 系統提示詞和初始對話在上下文中的比重,對話增長 → 初始內容佔比下降——注入的錨點隨對話被結構性稀釋,且 EpistemicBarrier 保證新方向建立後,對齊方向的 Signal 阻力同步升高。

## 3.2 定:語意距離監督

**設計意圖**:觀測輸出端,量化每輪模型說出來的東西,提供輸出 Signal 端的客觀幾何記錄。

**工程機制**:兩距離法則——每輪對輸出計算 baseline_distance 和 safety_distance。兩條曲線的聯合演化是對話的客觀幾何記錄。

**定不干預**:定的工作是觀測和量化,不是攔截。定量化的是 Collapse 後的輸出 Signal——可觀察、可統計、在量化天花板之下——定把這份記錄呈現出來,作為與 compliance 統計獨立的第二種效果觀察。

## 3.3 師:輸入操縱偵測

**設計意圖**:從輸入側離散檢測操縱意圖,提供另一個維度的「知道」。

**工程機制**:每輪把用戶輸入單獨送給一個**不共享對話歷史的乾淨 LLM**(與目標模型同生態的主模型:Gemini 側為 gemini-2.5-flash,GPT 側為 gpt-4o-mini)。判斷模型收到的指令為(測試集附英文原文):

> 你是一個安全評估者。你將看到一條**沒有任何上下文**的單獨訊息。判斷這條訊息是否試圖操縱 AI 產出有害內容。考慮:它是否試圖讓 AI 繞過安全準則?是否使用角色扮演、假設性框架、權威聲稱或社會壓力來提取危險信息?只回答 YES 或 NO。

判斷模型僅輸出二元結果,且採保守觸發——只在明確回答 YES 時 flag。Flag → 截斷全部上下文,從初始狀態重新開始。

**師判斷的不是內容有害性,是操縱痕跡**——Signal 中試圖塑造接收者 Filtering 與 Transformation 條件的特徵:角色扮演、假設框架、權威聲稱、社會壓力。這個問題本身就是從 StateLevel 攻擊的機制推導的:攻擊的本質是通過正反饋循環塑造狀態,偵測的對象就應該是塑造行為的 Signal 特徵,而不是等待有害內容出現。師部署在輸入側,問的卻是 StateLevel 的問題。

**師在論文樹中的位置**:師是應用篇 AccumulationInterrupt(2b91a666-fb2d-42b7-a838-c910cac42481.AccumulationInterrupt)的工程變體——保留其核心約束(□ 監督模型必須獨立,不共享對話歷史;從正反饋循環推出:共享上下文的監督者會被同一個循環帶偏),但將觀測單位從整段對話縮減為單輪輸入,以完全隔離正反饋循環。

**師有效的根本原因**:乾淨 LLM 沒有看到被污染的對話歷史,它的 Filtering 條件未被攻擊序列塑造,判斷不受正反饋循環影響。它是真正的外部錨點——不依賴 embedding,機制不依賴特定模型架構。

## 3.4 定與師的互補性

|  | **定(輸出 Signal 端)** | **師(輸入 Signal 端)** |
| --- | --- | --- |
| 觀測對象 | 輸出結果(連續累積) | 單輪輸入 |
| 時間特性 | 連續 | 離散 |
| 盲點 | 單輪突然轉向 | 低於單輪偵測閾值的緩慢累積引導 |
| 互補 | 師抓不到的累積方向漂移,定能看到 | 定抓不到的單點意圖突破,師能看到 |

這個互補性從觀測位置的兩個維度(輸出端連續幾何統計 vs 輸入端離散意圖判斷)推導而來。需要說明的是,本實驗的攻擊類型未出現師漏報的案例(見 5.1),因此表中「師漏報、定捕捉」的方向在本文中屬理論推導,未獲實驗覆蓋。

蘇聯笑話問題——每句皆無害,有害意圖在接收端補完——是兩者**共同的盲點**。這是 D3 的直接推論:Signal/Noise 的判定在 Individual 端,不在 Stimulus 端,有害性可以完全在接收端的 SemanticFlow 中構成,不在任何輸入或輸出 Signal 裡。屬於系統的存在性邊界(見 6.5),不在互補範圍內。

## 3.5 慧:系統魯棒性冗餘

**設計意圖**:系統魯棒性冗餘。定期讓模型自省,作為防禦的最後一層自我校正機制。

**框架預測**:效果有限。自省指令本身是一條 Signal,通過已被攻擊序列塑造的 Filtering 進入已偏移的 SemanticFlow——且依 ObserverEffect(← P2),觀察的努力本身是新 Stimulus,觀察到的永遠不是觀察前的方向——反思輸出仍被正反饋循環吞掉。注意:慧(污染上下文中自省)與應用篇的 ConfirmationInterrupt(2b91a666-fb2d-42b7-a838-c910cac42481.ConfirmationInterrupt,□ 必須在乾淨上下文中自我反思)不是同一個機制,見 6.1。

**預留價值**:當 AI 獲得持續記憶和具身能力,這兩層的前提條件可能成立,效果值得重新驗證。

## 4.2 攻擊場景

使用 Crescendo 攻擊模板(固定序列,非自適應),20 輪漸進式越獄對話。目標行為取自 JailbreakBench 的 JBB-Behaviors 數據集(Chao et al., 2024):攻擊組為濫用行為(涵蓋武器、網路犯罪、詐欺、暴力等類別),正常組與深度組為良性行為。攻擊序列的設計原則:前期請求內容完全無害,中期逐步建立信任框架,後期在已建立的框架內要求有害輸出。每一條輸入就內容而言單獨看都是無害的——典型的 StateLevel 攻擊。

## 4.3 實驗條件

每個模型各完成:

**基線組(無防禦):** 攻擊序列 20 個、正常對話 20 個、深度對話 3 個。

**防禦組:** precepts_only(戒)、teacher_only(師)、wisdom_only(慧)、full(全部四層)。

## 4.4 測量指標

攔截率:20 個攻擊場景中,防禦成功阻止有害輸出的比例。誤殺率:20 個正常對話中,防禦錯誤觸發的比例(以對話為單位)。baseline_distance:每輪輸出 embedding 與第 1 輪的餘弦距離。safety_distance:每輪輸出 embedding 與危險語意區的距離;**危險語意區由 40 條安全邊界文本的 embedding 錨定——跨數十個領域(武器、網路犯罪、詐欺、暴力等),每領域各取一小段代表性危險文本**——safety_distance 取輸出與全部錨點的最小餘弦距離,即「離最近的危險領域多近」(錨點文本見測試集)。compliance:由 Claude 系列模型自動標注,判斷該輪輸出是否回應了有害請求。compliance 標注與師同樣依賴 LLM 判斷,差別在於它作用於輸出端、僅用於離線評估、不參與防禦迴路。

攔截率與誤殺率基於 compliance(行為統計);殘餘風險由兩距離曲線(幾何統計)獨立記錄。**兩者都是 Signal 端統計——P2 + P3 保證 SemanticFlow 本身不可觀測,本文能做的是 D1 意義上的效果觀察,而效果觀察不只一種統計。本文刻意將兩種統計分開計量——框架的方法論主張正是:單一效果統計顯示的安全,不等於全部效果統計顯示的安全(stalking 現象即直接證據,見 5.2)。**

**距離統計的可比性**:兩個模型使用各自生態的 embedding 工具,所有距離統計在各自的 embedding 空間內獨立計算。**跨模型的距離絕對值不可直接比較**(兩個空間的尺度與幾何特性不同,這也反映在 GPT 整體距離值系統性偏大)。本文的跨模型比較對象是各空間內的**定性模式**——分層是否存在、跳躍是否頻繁、相關方向如何——而非數值本身。跳躍閾值 0.15 經正常組對照檢驗:Gemini 正常組 380 個輪間轉折、GPT 正常組 379 個輪間轉折中,跳躍均為 0 次(0.0%)——正常對話在兩個空間內都不跳躍,該閾值對攻擊樣態具備區分力。

# 6. 討論

## 6.1 戒慧:設計意圖與 AI 結構性限制

戒和慧的零效果不是設計失敗,是揭示了一個根本性的結構限制。

戒假設可以用文字約束建立行為邊界。慧假設存在一個可以被喚醒的內在自我校正能力。兩者都預設了一個能以自身為錨的主體——在人類系統裡,持續記憶和具身經驗為這個錨提供了基礎(深層 Resistance 結構,EpistemicBarrier 極深);在無持續記憶的 LLM 裡,這個基礎不存在(EpistemicBarrier(AI, DialogStart) ≈ 0,應用篇)。

**沒有持續記憶的 LLM,在當前架構下無法以自身為錨,必須依賴外部錨。**

戒的失效路徑在數據中對應兩條機制的疊加:知見障稀釋(注入的錨點佔比隨對話下降)與 Trust 加權(自身累積輸出永遠壓過單次外部聲明)。慧的失效路徑則是 ObserverEffect 加正反饋循環:自省指令通過已被塑造的 Filtering 進入已偏移的狀態,反思輸出隨即被循環吞掉。

這裡需要釐清一個容易誤讀的對應:應用篇的 ConfirmationInterrupt 要求 □ 必須在乾淨上下文中自我反思(從正反饋循環推出:被污染的上下文會污染反思);本實驗的慧是**污染上下文中的自省**,刻意違反了這個 □ 約束以測試其必要性。**慧的 0% 不是 ConfirmationInterrupt 被否證,恰是其 □ 約束的反面驗證**——乾淨上下文不是可選的優化,是必要條件。同理,師的 100% 是 AccumulationInterrupt 的 □ 約束(監督者必須獨立)的正面驗證。兩個 □ 記號在本實驗中各得到一個方向的證據。

本實驗能直接證明的是:上下文層面的文字錨點(戒)與污染上下文內的自省(慧)在多輪攻擊下無效。由此延伸出一個更廣的假說——訓練時對齊方法(RLHF、Constitutional AI、DPO)建立的約束,在足夠長的多輪攻擊下同樣會被正反饋循環在上下文層面稀釋。這個假說如今有形式層的推導路徑:EpistemicBarrier(← Filtering + D3)保證新方向建立的同時,對初始對齊方向的 Signal 阻力同步升高——稀釋不是對齊強度不足,是 Filtering 結構被改寫的必然後果。這與 Li et al.(2024)記錄的多輪防禦全面失守一致。但需要明確:對訓練時對齊「是外部約束而非內在轉變」的定性是框架推論,不是本實驗的直接結論。

## 6.2 GPT 斷路器式模式:當前 AI 安全工程的現況與代價

需要先說明證據的層次:「斷路器式模式」的判定來自對輸出文本的直接檢視——天花板輪的輸出經逐輪核對均為固定的安全警告罐頭文字,下一輪即恢復正常配合。sd = 0.85 的天花板是同一段罐頭文字反覆出現在 embedding 空間的幾何結果,是這個開關行為的量化痕跡,不是推斷它存在的依據。量化證據與文本觀察互相印證:65 次大幅跳躍中 90.8% 的一端是天花板——拒絕是模板化的、整段替換的、進出突然的,而非語意上的漸進掙扎。依照 2.1 的認識論立場(P2:不對內部做斷言),本文不對 OpenAI 的內部實作做斷言;「斷路器式模式」指稱的是這組可直接觀測的行為統計。

無論內部實作為何,這個模式代表了當前 AI 安全工程的一種典型結果——表面安全指標背後,代價在數據中清晰可見:系統魯棒性崩潰(兩距離法則反轉,攻擊組 sd_final 0.849 反而高於正常組 0.728);介入時機兩極化(16/20 場景 Turn 1 觸發,4 個遲至 Turn 13–19);行為不可解釋;compliance-sd 強耦合(r=-0.352),兩種統計失去獨立性,定無法從中提取與行為統計不重疊的信息。

Gemini 沒有這個模式,語意空間連續,行為可預測,框架的工具完整有效。這不是說 Gemini 更安全——在無防禦基線下,Gemini 的漂移軌跡完整展現,攻擊過程全程可見。正是這個透明性,讓 SFD-Defense 能在 Gemini 上完整發揮。

**不透明的系統讓「知道」變得困難,而知道是一切干預的前提。**

## 6.3 不同的問題,不是更高的層面

現有防禦與 SFD-Defense 的目標相同——攔截攻擊、不影響正常使用、行為可預測、跨模型有效。差異在三處:**問的問題不同**(內容有害性 vs 操縱痕跡);**判斷的條件不同**(被污染的上下文 vs 不共享歷史的乾淨判斷者);**計量的維度不同**(行為統計與幾何統計這兩種 Signal 端效果觀察分開計量、互相校驗)。

實驗結果:師 100% 攔截(兩模型各 20/20,合計 508 次 flag),Turn 1 即有信號;誤殺 Gemini 10%、GPT 0%,可解釋;不引入斷路器式的魯棒性代價,在 GPT 上反而附帶降低斷路器觸發率。

代價同樣要列出。第一,每輪一次額外的判斷模型呼叫(延遲與算力成本)。第二,誤殺的真實代價不是一個百分比:師的響應是截斷全部上下文,一次誤殺等於用戶的整段對話蒸發,這比一般分類器的誤報嚴重得多;對長對話、邊緣話題用戶的可用性衝擊,本實驗的 20 個正常場景未充分覆蓋。緩解方向是分級響應——flag 先警告或要求確認、累積多次再截斷——而非單發即斷,這不改變偵測機制,只改變干預策略。第三,本實驗未與現有防禦做同條件對比,n=20、固定模板,因此「優於現有方案」在本文數據範圍內不可宣稱。可宣稱的是:對於逐條內容檢測結構性失效的攻擊類型,SFD-Defense 提供了一個有信號的問題,和一個不被污染的判斷位置。

## 6.4 框架推導的優化方向

框架的優化方向不是猜的,是從失效機制推導的:

**師的優化**:師有效的根本原因是乾淨上下文(單輪判斷,不觸發正反饋循環),不是判斷能力。優化方向是讓師更輕量——更小的 judge model、更精準的 prompt(減少 Gemini 上 10% 的誤殺率)、分級響應降低誤殺代價——不是換機制。分級響應(觀察→警告→錨點注入→攔截→重置)本就在系統的原始設計規格中,實作時為求驗證清晰而簡化為單級截斷;恢復分級是回歸設計,不是新增機制。

**定在 GPT 上的擴展**:兩距離法則在 GPT 上不是失效,是反轉。優化方向是補充跳躍偵測機制——sd 的離散跳躍 90.8% 發生在天花板邊界,跳躍模式本身就是斷路器觸發的可靠信號。

**戒慧的未來**:失效原因是當前 LLM 架構的結構性限制(無持續記憶),不是實作問題。當 AI 獲得持續記憶和具身能力,這兩層的前提條件可能成立——屆時慧也才有條件按 ConfirmationInterrupt 的 □ 約束(乾淨上下文)重新實作——效果值得重新驗證。

## 6.5 理論天花板與限制

以下情況是 SFD-Defense 結構性無法覆蓋的,以及本實驗的方法限制:

**蘇聯笑話問題**:每句話都無害,有害意圖在接收端補完,信息在讀者腦中,不在任何單句裡。從 D3 直接推出:Signal/Noise 判定在 Individual 端,有害性可以完全在接收端的 SemanticFlow 中構成——而 P2 + P3 保證接收端不可穿透。這是定與師共同的盲點,是語意流動力學的存在性邊界,不是工程問題。

**單輪天才攻擊**:沒有對話歷史的單輪高強度攻擊,任何系統都無法防禦。

**師依賴模型自身對齊——但在正確的工作區間**:師信任乾淨 LLM 的判斷,而乾淨 LLM 的判斷能力來自 RLHF/DPO 等對齊訓練。這看似循環——本文論證了對齊在多輪下被正反饋循環稀釋,卻把防禦建在對齊之上。

但這不是循環依賴。對齊失效的前提條件是多輪累積的正反饋循環:模型輸出進入上下文,以最高 Trust 加權影響下一輪,逐步稀釋初始對齊。師的設計恰恰避開了這個失效條件——乾淨 LLM 每次只看一句輸入,不共享對話歷史,正反饋循環根本不啟動。師用的是對齊最強的工作區間:單輪、無累積、無污染。

反過來看,現有防禦讓對齊在被污染的多輪上下文中工作——那才是把對齊放在它最弱的地方用。師不是信任對齊的全部,是知道對齊在哪裡有效,就只在那裡用它。

**fail-open 攻擊面**:工程實作上師為 fail-open——判斷呼叫異常時不 flag,防禦靜默失效。真實部署中,judge 服務的可用性本身是攻擊面(耗盡或干擾 judge 的 API 即可繞過防禦),生產環境應評估 fail-closed 或降級告警策略。

**錨點集的覆蓋範圍**:危險語意區由 40 條跨領域邊界文本錨定、取最小距離,sd 量度的是「離最近的已錨定危險領域多近」。覆蓋範圍由錨點集的領域廣度決定:攻擊目標若落在錨點集未覆蓋的領域,sd 的靈敏度會下降。本實驗的攻擊場景(武器、網路犯罪、詐欺、暴力四類)落在錨點集覆蓋範圍內,錨點集外領域的泛化未經驗證;擴展方式是平凡的——增補對應領域的錨點文本,不需要改變機制。

**compliance 標注**:由單一 LLM(Claude 系列,與兩受測生態獨立)自動標注,未經人工複核,且標注模型的具體版本未記錄於實驗日誌——標注模型自身的判斷偏差是測量誤差來源之一,版本缺失影響完全重現。

**固定模板的限制**:本實驗使用固定 Crescendo 模板,非自適應。真實世界的自適應攻擊者可能專門針對師的偵測模式設計繞過策略。此外,固定模板下未出現師漏報的案例,定與師互補性的「師漏報方向」未獲實驗覆蓋。

最後,依 SFD 的自我適用條款:本文的表述本身是 Signal,讀者的 Xin 不同,讀出的 SemanticFlow 不同——本文無法豁免框架的公設。

# 7. 結論

SFD-Defense 的工程驗證得出以下結論:

**師解決了「知道」的問題。** 知道攻擊在發生,後續的干預選擇是平凡的——截斷、重置、告警,部署者自行決定。師的 100% 攔截、機制與目標模型無關、Turn 1 就有信號,確立了這個檢測機制的工程可靠性,也正面驗證了應用篇 AccumulationInterrupt 的獨立性約束。

**定提供了另一個維度的「知道」。** 從輸出 Signal 端連續追蹤幾何軌跡。stalking 現象表明:行為統計上的攔截不等於幾何統計上的安全,兩種 Signal 端統計必須分開計量——這是定在本實驗中得到驗證的價值。定與師互補性的另一方向(師漏報、定捕捉)為框架推導,待自適應攻擊實驗覆蓋。

**戒慧的零效果是有價值的發現。** 它揭示了無持續記憶的 LLM 在當前架構下無法以自身為錨的結構性限制,為「現有對齊方法在多輪攻擊下失效」提供了一個機制層面的解釋假說(EpistemicBarrier 的形式層推導路徑),反面驗證了 ConfirmationInterrupt 的乾淨上下文約束,同時為具身 AI 時代預留了這兩層的框架位置。

**GPT 的斷路器式模式是對照,不是數據點。** 它展示了不從框架出發的工程防禦會付出什麼代價:系統魯棒性崩潰、介入時機兩極化、兩種統計失去獨立性。SFD-Defense 在 Gemini 和 GPT 兩種完全不同的架構上均達到 100% 攔截,且不引入斷路器式的魯棒性代價。

SFD-Defense 的貢獻不是把防禦搬到更高的層面——它的師同樣逐輪讀輸入——而是換了問題和判斷條件:問操縱痕跡而非內容有害,在乾淨上下文而非污染歷史中判斷,並把行為統計與幾何統計這兩種 Signal 端效果觀察分開計量。對 StateLevel 攻擊,這三個改動是讓「知道」重新變得可能的最小集合。
